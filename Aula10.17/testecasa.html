O que é Git?

  • É um sistema de controle de versão, como Subversion, CVS(Concurrent Version System) e Perforce, mas com características bem diferentes destes
  • Em geral se pode trabalhar com o git mesmo sem acesso a outros computadores. Todo o histórico de seu projeto fica local
  • Arquivos no git podem estar em 3 estados:
  • Committed: dado guardado com segurança em seu repositorio local
  • Modified; você mudou seu arquivo, mas não o enviou para seu repositorio
  • Staged: você marcou um arquivo modificado para que vá no próximo commit

Seções de um projeto Git

Working directory Checkout (versão baixada) de uma versão do projeto Esses arquivos são trazidos do banco de dados comprimido no diretório Git e colocados no diretório para que você trabalhe com eles

Staging Area É um único arquivo, normalmente no seu diretório Git, que armazena dados sobre o que irá no seu próximo commit.

Git directory (repositório) Local onde o Git armazena os metadados e banco de dados do seu projeto
